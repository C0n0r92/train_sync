## Bugs Found During Testing

### Bug #1: JWT Secret Key Error (FIXED)
**Discovered**: Auth controller initialization
**Root Cause**: Using deprecated `Rails.application.secrets.secret_key_base` (Rails 5 syntax)
**Solution**: Changed to `Rails.application.config.secret_key_base` (Rails 6+ syntax)
**Impact**: Would have blocked all JWT generation
**Status**: ✅ FIXED

### Bug #2: Empty Blocks Array Parsing (FIXED)
**Discovered**: Workout creation test with empty blocks
**Root Cause**: Rails parameter permit with `blocks: []` treats empty form input as `[""]` instead of `[]`
**Solution**: Added explicit normalization: `permitted[:blocks] = [] if permitted[:blocks].blank? || permitted[:blocks] == ['']`
**Impact**: Causes validation issues and incorrect JSONB storage
**Status**: ✅ FIXED

### Bug #3: Double Render in Authorization (FIXED)
**Discovered**: Draft workout authorization test
**Root Cause**: Authorization methods render error response but don't return, allowing action to continue and render again
**Solution**: Added explicit return statements in auth methods
**Impact**: Rails error, request fails with internal error instead of proper 403 response
**Status**: ✅ FIXED

### Bug #4: Generic Error Messages (FIXED)
**Discovered**: Signup validation tests
**Root Cause**: Error responses only returned generic "Signup failed" message
**Solution**: Parse error messages from model validations and include field-level details
**Example**: "password_confirmation: doesn't match Password" instead of generic error
**Impact**: Poor API consumer experience, debugging difficult for frontend
**Status**: ✅ FIXED

## Testing Approach & Lessons Learned

### What Worked Well
- **Test-Driven Development**: Writing tests FIRST uncovered 4 bugs before implementation
- **Comprehensive Test Plan**: PRD-based test cases ensured all requirements were covered
- **Factory-Based Test Data**: Clean fixtures made tests maintainable and focused
- **Explicit Edge Cases**: Tests for expired tokens, duplicate emails, authorization failures caught real issues

### Insights From Bug Detection
1. **JWT/Config Bugs** - Always verify Rails version specific APIs
2. **Parameter Parsing** - Rails form parameter handling has unintuitive edge cases with empty arrays
3. **Render Logic** - Authorization checks MUST include return statements, even in Rails callback context
4. **Error Reporting** - Generic error messages violate API best practices; always expose validation details
